Production Steps

This section outlines the production process of the IPAX. Though not necessarily needed to be done in this particular order, we recommend users to build an IPAX in the step as shown in the diagram below. This page provides steps for assembly process, but the detail of each step can be found in the respective page.

1. Procurement

The entire bill of materials for the IPAX is provided in the BOM folder. The BOM also indicates recommended vendors, though users can procure their components through any vendors.

2. Printed Circuit Boards

There are 3 printed circuit boards in the IPAX that need to be fabricated. The KiCAD files for all three boards are available in the PCBs folder. In each folder for each PCB, there is a folder called "Gerber" that contains gerber files that are ready to be sent to a PCB house for fabrication.

The control board and the power regulator boards can be fabricated with regular FR-4 substrate, but we strongly recommend that the LED array board should be fabricated with aluminum substrate to help with heat sink.

*There is an optional 4th board called "optics holder" that is not required, but we recommend if users plan to use the IPAX for an application that requires the LED to be on for an extended period of time. This board mainly helps attach the LED array board to the aluminum flange of the housing. It will help dissipate heat from the LEDs out very quickly. However, if heating is not a problem, it can be replaced with a simple 3D printed part.

**Another optinal PCB is an SD card add-on, which will be deprecated and added directly to the control board in future revisions. However, for now, it provides the control board with an access to an SD card without the need to turn the Pi on. The SD card can be used to store environmental data and also to store configuration files.

3. 3D Printed Parts

There are 3 components that need to be printed out using an FDM 3D printer. The STEP files of all three components are provided in Mechanical folder, along with the STEP file for the entire assembly. Note that we use Fusion 360 as the CAD software, so there might be some issues if the files are opened with other software.

There are no special requirements for all these 3D printed parts. We use PLA material to print all of them. Note that the two battery holders shown in the picture below are identical.

*Optics holder can be replaced with an aluminum PCB for better heat dissipation.

4. LED and Camera Assembly

The LED board and the Arducam camera board can be attached to the optics holder PCB (or 3D printed part) using a set of M3 screws and standoffs as shown in the image below. The assembly can then be attached to the aluminum flange using a set screw and a screw.

Note that depending on the M12 lens, the position of the camera might need to be adjusted. The provided files are suitable for M12 lens with focal length no longer than 16mm; otherwise, the lens would be too long to fit inside the housing. The lens should be almost flat on top of the acrylic viewport to prevent the LED light from reflecting back into the camera lens.

5. PCB Tray Assembly

The Pi Zero W can be attached to the 3D printed PCB tray with a set of M2.5 standoffs (M3 does not fit through the holes on the Pi zero board). Note that this is the only place where M2.5 screws are used; the rest of the system uses M3 only.

The control board is attached directly on top of the Pi with the 40-pin GPIO headers. The power regulator board is attached to the other side of the PCB tray. Then, the entire tray is attached to the flange. Note that the bottom of the tray (the power regulator side) can be reinforced with 87mm M3 standoff (combination of multiple standoffs).

6. Battery and End Cap Assembly

The battery used in this tutorial is Tenergy 31019. Other batteries will work as well, but users need to modify the 3D file for the battery holder.

The endcap requires at least two switches: one for main power and the other for shutdown signal. The environmental sensors are optional and can be replaced with blank screws. All wire-to-wire connections are done with Molex Minifit connectors.

Note that we do not recommend Blueorobotics vent plug, and its function can be done through a switch instead. See mechanical section for more information.

7. Programming

Programming on the control board is done with Arduino platform and on the Pi Zero W is done with Python. The detail can be found on programming section.
